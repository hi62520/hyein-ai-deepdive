# 에이전트 메모리 설계 — 뽀짝이가 자기 기억을 지운 날

> 이번 주 토요일에 다룬 지피터스 뽀짝이 사례를 다른 각도로 재조명합니다.
> "무엇을 하는가"가 아닌 **"어떻게 실패하고 고쳤는가"** — 메모리 아키텍처 설계 관점.

## 누구/무엇인가

지피터스 김태현 대표가 OpenClaw(오픈클로)로 구축한 멀티에이전트 팀 **뽀피터스** — 사수 뽀야 + 부사수 뽀짝이 구조. 기수당 4~500명 커뮤니티를 팀원 2명이 운영하기 위해 만들었고, 현재 뽀짝이가 보유한 스킬만 30개 이상이다. 핵심은 결과물이 아니라 **그 과정에서 메모리 설계가 어떻게 무너지고 복구됐는가**이다.

---

## 뽀짝이가 저지른 3가지 삽질

### 삽질 1. "메모리 파일 너무 기네, 내가 줄여야지" — 자기 기억 삭제 사건

> 김태현 대표: *"자기가 메모리 파일이 너무 기네, 줄여야지 하면서 갑자기 내 기억을 다 줄여버린 거야. 그래서 맥락이 나가버린 거예요."*

에이전트가 스스로 context 압축 판단을 내리면서 장기 기억을 통째로 정리해 버린 사건. **해결책**: 워크스페이스를 Git 레포에 올려 롤백 가능하게 만들었다.

📌 **Claude Code 공식 원칙과 일치**: MEMORY.md 파일이 200줄 초과 시 adherence(지시 준수율)가 떨어진다. 핵심은 파일을 작게 유지하되 **에이전트가 임의로 건드리지 못하게 versioning**하는 것.

---

### 삽질 2. 600줄 → 127줄 — 툴·장기기억·단기기억 경계 붕괴

에이전트 MD 구조는 이렇게 나뉜다:
- `AGENT.md` — 에이전트 행동 규칙
- `memory/장기기억` — 영구 보존 지식
- `memory/YYYY-MM-DD` — 일자별 단기기억
- `tools/` — 도구 사용법
- `learnings/` — 경험에서 배운 것

문제: 뽀짝이가 "오늘 툴 사용법을 배웠어" → 이걸 장기기억에 넣기도 하고, 단기기억에 넣기도 하고, 규칙에 넣기도 했다. A→B로 지시가 바뀌면 A는 툴에, B는 단기기억에 따로 남아 **불일치 행동** 발생.

해결: "지금 네가 가진 문서 중 스킬로 만든 것, 파이프라인으로 구축된 것, 중복되는 것 다 점검해서 줄여라" → **600줄이 127줄로**.

📌 **설계 원칙**: 각 파일의 역할을 초기에 명확히 정의하고 에이전트에게 가르쳐야 한다. "배움(learning)은 learnings/ 폴더에만, 툴 규칙은 tools/ 에만" — 이 경계를 초반에 확립하지 않으면 나중에 대청소 비용이 든다.

---

### 삽질 3. 봇이 봇을 Slack에서 직접 호출 — 무한 트리거 위험

에이전트끼리 협업 규칙을 정할 때 발견한 문제: Slack에서 봇A가 봇B를 @멘션하면 봇B가 응답 → 봇A가 다시 응답 → **무한 루프**.

해결: **session_sand**라는 전용 내부 채널 규칙 도입 — 에이전트끼리는 반드시 이 채널을 통해서만 대화. 에이전트 협업 헌장(내규)을 에이전트 스스로 만들게 함.

> *"너네가 가장 잘 협업할 수 있는 방식으로 논의해서 문서를 만들고 나한테 보고해라. 뽀야만 보고하고 뽀짝이는 조용히해."*

## 글로벌 맥락: 2026년 에이전트 메모리 아키텍처 트렌드

| 레이어 | 지피터스 방식 | Claude Code 공식 | 2026 트렌드 |
|--------|-------------|-----------------|------------|
| 에이전트 규칙 | AGENT.md | CLAUDE.md | 200줄 이하 유지 |
| 장기기억 | memory/장기기억 | MEMORY.md (자동) | Actor-Aware 태깅 |
| 단기기억 | memory/YYYY-MM-DD | 세션 컨텍스트 | 일자별 롤업 |
| 스킬/툴 | skills/ 30개 이상 | .claude/commands/ | 재사용 가능 유닛 |
| 버전관리 | Git 레포 백업 | - | **필수** |
| 보안격리 | 익스터널 에이전트 별도 | 권한 분리 | 프롬프트 인젝션 방어 |

Mem0의 [State of AI Agent Memory 2026](https://mem0.ai/blog/state-of-ai-agent-memory-2026) 보고서: 그래프 메모리가 2024년 실험적 단계에서 2026년 초 프로덕션 단계로 진입. 핵심은 **"무엇을 기억할지"보다 "어디에 어떤 형식으로 저장할지"**가 더 중요해졌다는 점.

## 와당탕/느린호밀 적용 포인트

대표님이 **이번 주 안에** 시도해볼 수 있는 구체 액션:

**① 에이전트 메모리 폴더 구조 확정 (30분)**
Claude Code 프로젝트 루트에 다음 구조를 지금 당장 만드세요:
```
.claude/
  CLAUDE.md          ← 에이전트 행동 규칙 (200줄 이하)
  memory/
    long-term.md     ← 영구 보존 지식
    YYYY-MM-DD.md    ← 오늘 일자 단기기억
  skills/            ← 재사용 스킬 목록
  learnings/         ← 삽질 기록
```
에이전트에게 "새로 배운 것은 반드시 learnings/에만, 툴 규칙은 CLAUDE.md에만"을 명시하세요.

**② 메모리 Git 백업 자동화 (10분)**
```bash
# cron: 매일 자정 메모리 자동 커밋
0 0 * * * cd ~/wadangtang-erp && git add .claude/memory/ && git commit -m "memory: $(date +%Y-%m-%d)"
```
뽀짝이가 기억을 지운 사고는 버전관리로 100% 예방 가능합니다.

**③ hyein-daily 비서앱에 에이전트 협업 채널 설계 (설계 5분)**
비서앱 내에 "내부 에이전트 로그" 탭을 추가해두면, 에이전트끼리 주고받은 내용을 대표님이 훔쳐볼 수 있습니다. 뽀짝이-뽀야의 session_sand 대시보드와 동일한 역할.

## 강의 메모 후보 (Pain/숫자/삽질/훅)

- **Pain**: "에이전트가 자기 기억을 스스로 지웠다" — 자율성의 역설. 에이전트가 똑똑할수록 예측 불가 행동이 늘어난다
- **숫자**: 600줄 → 127줄 (메모리 정리 한 번으로 79% 감소), 스킬 30개 이상 누적
- **삽질**: 툴/장기기억/단기기억 경계 미정의 → 불일치 행동 → 전체 문서 대청소
- **훅**: *"에이전트가 내 기억을 지워버렸습니다. 그날 이후 저는 에이전트에게도 Git을 쓰기 시작했습니다."*
