# Tony Dinh — 제품이 팀이다: Agent Skills 내장과 B2B 피벗으로 월 1.5억 1인 기업

## 누구/무엇인가

Tony Dinh(@tdinh_me)는 베트남계 싱가포르 기반 솔로 인디 해커입니다. 빅테크 소프트웨어 엔지니어 직을 그만두고 풀타임 1인 창업자가 된 그는, 2023년 3월 OpenAI가 ChatGPT API를 공개한 날 하루 만에 TypingMind의 첫 버전을 빌드해 출시했습니다. TypingMind는 "API 키를 직접 들고 오는" 방식의 프리미엄 LLM 채팅 인터페이스로, 출시 첫 주에만 $22,000을 벌었습니다. 2026년 현재 월 $137K MRR(약 1.6억 원), 누적 매출 $1.6M ARR을 기록하며 Fortune 500 기업 고객을 보유한 기업용 AI 워크스페이스로 진화했습니다. 팀 규모는 여전히 3명 이하입니다. TypingMind 외에도 DevUtils($5K/월), makeawebsite.app(신규), Black Magic($128K 매각), Xnapper($150K 매각)까지, 그는 "만들고-성장시키고-팔거나-유지한다"는 포트폴리오 방정식을 몸소 구현하고 있습니다.

## 무엇이 특별한가

### 1. B2B 피벗 — 개인 고객을 넘어 Fortune 500으로

초기 TypingMind는 ChatGPT 파워유저 대상의 $39 일회성 구매 제품이었습니다. Tony는 기업 문의가 늘자 **TypingMind Teams/Custom** 라인을 추가하며 B2B로 피벗했습니다. 2025년 말 기준 전체 MRR 중 B2B 구독이 50%를 초과했고, SSO·RBAC·감사 로그·SOC 2 Type II·HIPAA·GDPR 준수 기능을 탑재해 3,000석짜리 엔터프라이즈 계약을 따냈습니다. Tony의 회고: *"나는 기업을 상대하기 위해 영업팀을 고용한 게 아니라, 제품이 스스로 기업에게 팔리도록 만들었다."* 즉, 세일즈를 사람이 아닌 셀프서브 온보딩과 기능 설계로 대체한 것입니다.

### 2. Agent Skills 내장 — UI에서 에이전트 런타임으로

2026년 6월 1일, Tony는 **Agent Skills** 기능을 출시했습니다. Skills는 `SKILL.md` 파일을 중심으로 구성되는 태스크별 절차 지식 묶음으로, SEO 감사·카피라이팅·코드 리뷰·브레인스토밍 등 역할별 전문 능력을 AI에 심는 오픈 표준입니다. 사용자는 TypingMind 내 Plugins → Skills Store에서 설치하거나, GitHub URL 또는 ZIP으로 자신만의 스킬을 임포트할 수 있습니다. 이는 TypingMind를 단순 "예쁜 ChatGPT 창"에서 **조직 전체의 에이전트 실행 환경**으로 격상시키는 전환점입니다. 같은 달 Product Hunt 재런칭(3년 만)을 통해 신기능을 노출시켰고, Tony는 이를 두고 *"Product Hunt은 6개월마다 해야 했는데, 처음 3년간 그러지 않았다. 쉬운 마케팅을 스스로 막았던 것"*이라고 자기비판했습니다.

### 3. 포트폴리오 매각 전략 — 팔 수 있을 때 팔고, 복리는 남긴다

Twitter API 요금이 3배로 오르며 Black Magic 운영 비용이 수익의 3배를 초과하자 Tony는 미련 없이 $128K에 매각했습니다. Xnapper(스크린샷 앱)도 $150K에 팔았습니다. 이 패턴의 핵심은 **플랫폼 리스크가 생긴 제품은 현금화하고, 복리 구조가 살아있는 제품(TypingMind)에 집중**한다는 것입니다. 현재 Tony의 포트폴리오는 TypingMind(주력)·DevUtils(캐시카우)·makeawebsite.app(신규 탐색)으로 구성됩니다. 그는 2025년 7월 70개 이상의 인디 해킹 주제를 담은 **인디 북**을 출간해 지식 자산화도 병행하고 있습니다. *"지금 TypingMind 수익이면 일을 완전히 그만두고 투자 이자로만 살 수 있다. 하지만 나는 빌딩 자체가 좋다."*

### 4. 1인 운영 인프라 — '팀'이라는 환상 없이도 돌아간다

Tony의 운영 공식은 간단합니다: 개발자 1명(본인) + 기본 디자인 + 스트라이프 + 메일박스. 고객 지원은 제품 자체가 최대한 셀프서브 가능하도록 설계해 지원 부하를 줄이고, 2025년 이후 팀원 2명 내외를 최소 인원으로 추가했습니다. 매월 뉴스레터(news.tonydinh.com)에 수익·코드·여행을 공개하는 "빌딩 인 퍼블릭"이 콘텐츠 마케팅을 자동화합니다. X(Twitter) 팔로워 181K+는 신제품 검증과 배포 채널을 동시에 해결합니다.

## 와당탕/느린호밀 적용 포인트

**오늘 30분 (micro):**
비서앱 또는 Obsidian에서 가장 반복되는 작업 하나(예: 미팅 요약, 주문 확인 회신, 콘텐츠 아이디어 생성)를 골라 `SKILL.md` 형식의 인스트럭션 파일로 초안을 작성합니다. 형식은 단순합니다: 태스크 설명 + 단계별 절차 + 예시 1개. Claude Code에서 이 파일을 `CLAUDE.md`의 도구 섹션에 붙이면 즉시 재사용 가능한 에이전트 스킬이 됩니다. (= frontmatter `micro_action`과 동일)

**이번 주 1-2시간 (mid):**
ERP 또는 비서앱의 반복 작업 중 하나를 "셀프서브 온보딩" 방식으로 설계합니다. Tony가 기업 고객을 영업 없이 유치한 핵심은 제품 내부에 온보딩 흐름을 완결 시킨 것입니다. 와당탕 ERP에서 신규 클라이언트가 처음 접속했을 때 3단계 이내로 첫 가치를 경험하도록 플로를 리팩터링하고, Claude Code로 온보딩 체크리스트 컴포넌트를 구현합니다.

```javascript
// 예시: 첫 로그인 시 온보딩 진행 상태를 저장하는 간단한 훅
function useOnboardingProgress(userId) {
  const steps = ['profile_set', 'first_order', 'first_report']
  const completed = steps.filter(s => localStorage.getItem(`onboard_${userId}_${s}`))
  return { completed, total: steps.length, percent: (completed.length / steps.length) * 100 }
}
```

**이번 달 실험 (macro):**
느린호밀 콘텐츠 중 가장 반응이 좋았던 주제 3개를 골라 Tony의 인디 북 전략처럼 **경험 기반 지식 상품** 설계를 시도합니다. 구체적으로: (1) 주제 선정 → (2) 3개월 내 강의 스크립트 초안 → (3) 무료 뉴스레터 연재로 베타 독자 반응 측정. 측정 지표: 오픈율 30%+, 링크 클릭률 5%+, 직접 문의 1건 이상.

## 한국 솔로 운영자 맥락에서 주의

**플랫폼 종속 리스크를 먼저 진단하세요.** Tony가 Black Magic을 팔게 된 이유는 Twitter API 가격 정책 하나였습니다. 와당탕 비서앱이나 ERP가 특정 외부 API(카카오 알림톡, 네이버 커머스 등)에 핵심 기능이 묶여 있다면, 요금 인상 또는 정책 변경 시 Tony와 똑같은 상황이 됩니다. B2B 계약을 확장하기 전에 플랫폼 리스크 맵을 먼저 그려두는 것이 현명합니다.

**"Fortune 500 B2B 피벗"은 한국 소규모 B2B에서도 유효하지만 인증 비용이 걸림돌입니다.** SOC 2 Type II·HIPAA는 한국 시장에서 ISO 27001·ISMS-P로 대응해야 하며, 인증 취득에 수개월 + 수천만 원이 소요됩니다. 대신 "공문 기반 검토 요청 대응"과 "개인정보처리방침 강화"로 단기 신뢰를 확보하는 전략이 현실적입니다.
