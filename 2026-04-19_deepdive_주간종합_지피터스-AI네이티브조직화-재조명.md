# 지피터스 김태현 — "AX는 사람이 병목이라 잘 안 된다"

> 이번 주 토요일 뽀짝이 사례를 **다른 각도**로 재조명합니다.  
> 어제는 "메모리 설계"(기술). 오늘은 "AI-Native 조직 전환"(운영·문화).

## 누구/무엇인가

김태현 대표(지니파이/지피터스)는 기수당 400~500명짜리 AI 스터디 커뮤니티를 팀원 단 2명 + AI 에이전트로 운영한다. 월 15만명이 방문하는 학습 커뮤니티를 키운 소프트웨어 프로덕트 빌더 출신. 그의 현재 실험 과제는 하나다: **전 직원이 Claude Code를 메인 업무 운영 시스템으로 쓰는 AI-Native 조직**.

---

## 무엇이 특별한가

### 핵심 진단: "AX는 사람이 병목이라 잘 안 된다"

> *"최대한 사람이 뭔가를 안 배우고 에이전트가 옆에 붙어서 그 친구가 일을 많이 해주면, 사람이 AX 되기 위해 뭔가 배워야 하는 거를 줄여주는 방향으로."*

직원들에게 Claude Code를 쓰라고 말만 해봤자 안 된다. 이유는 기술 거부감이 아니다. **데이터 접근권과 SSOT가 없어서 쓸 수가 없는 것**이다.

### AI-Native 조직 전환 5가지 선행 조건

**① 토큰 무제한 지원**  
팀원이 토큰 비용을 계산하며 Claude를 쓰면 실험이 멈춘다. 김 대표는 Claude Code Max Plan을 팀원 수만큼 지원한다. *"오늘 200만 토큰 써도 돼"* — 이 한마디가 뽀짝이 같은 AI 에이전트를 탄생시킨 출발점이었다. 인건비 대비 월 $200짜리 구독료는 비교 자체가 안 된다.

**② SSOT (단일 맥락 원천) 구축**  
지피터스는 Notion을 버리고 **Linear에 모든 맥락**을 집중했다. 말로만 "Linear에 넣어라"고 해봤자 팀원들이 안 했다. 그런데 Claude Code 안에서 일을 시작하니까 팀원 스스로 "에이전트가 내 맥락을 알려면 어디에 어떻게 기록해야 하지?"를 고민하게 됐다. SSOT는 지시가 아니라 **에이전트를 쓰면서 자연스럽게 만들어진다**.

**③ 스킬 생태계 공유 시스템**  
한 팀원이 만든 Claude Code 슬래시 커맨드(스킬)를 사내 AI 툴킷 사이트에 올리면, 다른 팀원이 바로 다운받아 쓴다. *"이 스킬 우리 툴킷 사이트에 올려줘"* 한 마디면 배포 완료. 개발자는 기능 개발이 아닌 **스킬 공유 인프라**를 구축하는 역할로 재정의됐다.

**④ 데이터 접근권·API 연결이 먼저**  
스터디 운영 데이터가 Google Sheets에, 매출이 별도 서비스에, 멤버십이 또 다른 곳에 흩어져 있으면 에이전트가 일을 할 수가 없다. 제로베이스라면 **자산화부터 시작**이다. Airtable DB 세팅, API 연결 — 이 인프라가 없으면 에이전트는 그냥 대화 봇이다.

**⑤ "퇴근 인터뷰" — 에이전트가 팀원의 AI 장벽을 파악한다**  
다른 팀원이 만든 에이전트 '자비스'가 매일 퇴근 시 팀원에게 묻는다: *"오늘 내가 한 일 중 왜 Claude Code 안에서 못 했는지?"*. 이 데이터를 모아보면 어떤 외부 서비스, 어떤 데이터 연결이 빠진 것인지 정확히 보인다.

### 2026년 2월의 변곡점: 오픈클로 창업자, OpenAI 합류

뽀짝이의 기반인 오픈클로(OpenClaw) 창업자 Peter Steinberger가 2026년 2월 OpenAI에 합류해 차세대 개인 에이전트 개발을 이끌게 됐다. OpenClaw 자체는 오픈소스 재단으로 유지. 합류 직전 GitHub 스타 14만5천개, 1주일 방문자 200만명을 기록한 프로젝트였다. **오픈클로 생태계는 창업자 없이도 계속 성장 중**이다.

## 와당탕/느린호밀 적용 포인트

대표님이 **이번 주 안에** 시도해볼 수 있는 구체 액션:

**① SSOT 파일 하나 만들기 (30분)**  
`/home/user/wadangtang-erp` 프로젝트 루트에 `PROJECTS.md` 생성:
```markdown
# 현재 진행 중인 프로젝트
- 비서앱(hyein-daily): [상태] [우선순위]
- ERP(wadangtang-erp): [상태] [우선순위]

# 이번 주 맥락
...
```
Claude Code가 이 파일을 참조하도록 `CLAUDE.md`에 명시. 에이전트가 맥락 없이 동문서답하는 문제를 한 번에 해결한다.

**② 반복 작업 3개를 슬래시 커맨드로 (15분)**  
`.claude/commands/` 폴더에 자주 쓰는 패턴 저장:
```bash
# 예: /daily-brief
매출 현황, 오늘 해야 할 일, 미완료 이슈를 요약해줘.
```

**③ "오늘 AI로 못 한 일" 노트 1주일치 쌓기**  
퇴근 인터뷰 대신, 오늘부터 옵시디언 Daily Note에 한 줄씩 기록:  
`AI 못 한 일: [이유]` → 한 주 후 패턴이 보이면 그것이 다음 자동화 과제다.
