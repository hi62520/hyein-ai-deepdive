# 우아한형제들 — 배민이 AI를 실무에 녹이는 방식

## 누구/무엇인가

우아한형제들은 국내 최대 음식 배달 플랫폼 배달의민족(배민)을 운영하는 한국 테크 기업입니다. 2010년 설립, 2019년 독일 딜리버리히어로(Delivery Hero)에 약 4조 7천억 원에 인수됐으며, 2025년 기준 임직원 약 2,154명 규모입니다. 주목할 점은 단순히 "AI를 쓰는 회사"가 아니라, 조직 전반에 AI 에이전트를 내재화해 실제 업무 흐름을 바꿔가고 있다는 것입니다. 2025년 4월에는 사내 개발자·기획자·디자이너 20여 명이 직접 쓴 실전 노하우를 엮은 책 《요즘 우아한 AI 개발》(골든래빗, 2025)을 출간하며 업계 주목을 받았습니다. 배민은 이제 사내 AI 에이전트 플랫폼 '물어보새(Murobosae)'를 통해 전사 구성원 33%가 AI를 업무에 쓰는 회사로 진화했습니다.

## 무엇이 특별한가

### 1. 《요즘 우아한 AI 개발》 — 코드보다 삽질이 더 많다

2025년 4월 출간된 이 책은 PR 홍보물이 아닙니다. 20여 명의 배민 실무자가 직접 GPT와 부딪히며 겪은 시행착오를 그대로 담은 현장 보고서입니다. 5개 파트 구성: ① AI로 개발 생산성 높이기 ② AI로 더 편리한 서비스 만들기 ③ AI로 쉽고 빠르게 데이터 활용하기 ④ 안정적인 AI 서비스 운영하기 ⑤ 로봇과 머신러닝 모델 최적화. 특히 "기존 머신러닝 기반 시스템에 생성형 AI 더하기"나 "코드 없이 데이터를 추출하는 AI 데이터 분석 봇 만들기" 같은 챕터는 수천 명 규모 서비스에서 실제 검증된 레시피입니다. 솔로 빌더가 읽어야 하는 이유는 단 하나 — 대기업의 삽질을 공짜로 흡수할 수 있기 때문입니다.

### 2. 메뉴뚝딱AI — "32%의 고민러"를 위한 GPT 추천 엔진

배민 데이터에 따르면, **앱 사용자의 32%는 오늘 뭘 먹을지 결정하지 않은 상태로 앱을 열었습니다.** 이 수요를 잡기 위해 만들어진 게 메뉴뚝딱AI입니다. 기술 핵심: 수천만 개의 리뷰 텍스트에서 맛·식감·함께 먹는 상황·기분 키워드를 추출해 GPT에게 넘기고, GPT는 추천 가게 목록의 제목과 설명을 생성합니다. 개인 정보 보호를 위해 리뷰 내용 중 사람 이름·전화번호 등은 숫자로 치환 후 처리. 런칭까지 약 3개월이 걸렸고, 1,000명 대상 튜링 테스트(이해도·매력도·적합도 3가지 기준)에서 AI 생성 문구가 사람 작성 문구를 앞섰습니다. 이 한 사례에서 "RAG + 프롬프트 엔지니어링 + 개인정보 처리 + 품질 평가 프레임워크"가 전부 들어 있습니다.

### 3. AI API 게이트웨이 — 멀티 LLM 공급자를 한 레이어로

배민은 AWS Bedrock, Azure OpenAI, GCP Imagen 등 **여러 AI 서비스를 동시에 씁니다.** 각 벤더마다 Quota Limit·Rate Limit·폴백 로직을 각 팀이 따로 구현하면 중복 작업과 보안 구멍이 생깁니다. 그래서 AI플랫폼팀이 직접 내부 AI API 게이트웨이를 만들었습니다. 이 게이트웨이는 ① 모든 AI 호출을 한 곳에서 관리(보안·비용 추적) ② 리소스 풀링으로 Rate Limit 우회 ③ HTTP 직접 호출 + ML SDK 두 방식 지원 ④ 서비스 개발자가 비즈니스 로직에만 집중할 수 있게 인프라 추상화 — 를 한 번에 해결합니다. 솔로 빌더 입장에서 이 구조는 "앤트로픽·오픈AI 동시 연동 + 비용 추적 + 폴백"을 하나의 래퍼로 묶는 개념과 동일합니다.

### 4. 물어보새 — 전사 33% 에이전트 사용 달성

배민 사내 AI 에이전트 플랫폼 이름은 **'물어보새(Murobosae)'**입니다. 2025년 말 기준 전체 구성원의 33%가 업무에 활용 중이며, 에이전트 답변에 대한 긍정 반응률은 **95%**, 조직별 문의 해결 속도는 **평균 21% 단축**됐습니다. 단순 챗봇이 아니라 "환경을 인식하고 목표를 향해 판단·행동하는 자율형 AI 시스템"으로 정의합니다. 이를 만들고 운영하는 사내 직원 역할을 **'AI 생산성 탐험가'**라고 명명. 배민의 AI 전환 4단계 로드맵: (1) 기본 AI 도입 → (2) 업무 자동화 → (3) 구조적 상용화 → (4) 비즈니스 코파일럿. 2025년 말 기준 2단계 완료 중, 2026년 3단계 진입을 목표로 하고 있습니다.

### 5. 다국어 지원·로봇 AI — 서비스 전방위 AI 내재화

LLM 기반 생성형 AI를 적용해 배민 앱에 영어·중국어·일본어 지원을 추가했습니다. 배달 로봇의 실외 자율주행을 돕는 ML 모델도 자체 최적화 및 MLOps 구축 경험을 쌓았습니다. AI 이미지 검수 시스템으로 저해상도·저작권 침해 가능 이미지를 자동 판별합니다. 이 정도 범위의 AI 내재화는 한국 테크 기업 중에서도 드문 케이스입니다.

## 와당탕/느린호밀 적용 포인트

**오늘 30분 (micro)**: 비서앱에서 '물어보새' 구조를 참고해 내부 FAQ 에이전트 1개 설계. 자주 묻는 질문 5개 목록 작성 후 Claude에게 답변 초안 뽑기. 이것이 물어보새의 가장 작은 버전입니다.

**이번 주 1-2시간 (mid)**: 비서앱의 여러 LLM 호출(Claude + 다른 API 혼용 시)을 위한 미니 게이트웨이 함수 만들기. 아래 구조로 시작하면 됩니다.

```python
# 미니 AI 게이트웨이 — 배민 아키텍처의 솔로 버전
import anthropic
import openai
from functools import wraps

def ai_gateway(provider="claude", fallback="gpt4o"):
    """단일 진입점 — 공급자 전환·폴백·비용 로깅 통합"""
    def decorator(fn):
        @wraps(fn)
        def wrapper(*args, **kwargs):
            try:
                result = fn(*args, provider=provider, **kwargs)
                log_cost(provider, result.usage)
                return result
            except RateLimitError:
                return fn(*args, provider=fallback, **kwargs)
        return wrapper
    return decorator

@ai_gateway(provider="claude", fallback="gpt4o")
def call_llm(prompt, provider="claude", **kwargs):
    if provider == "claude":
        client = anthropic.Anthropic()
        msg = client.messages.create(
            model="claude-sonnet-4-6",
            max_tokens=1024,
            messages=[{"role": "user", "content": prompt}]
        )
        return msg.content[0].text
```

## 한국 솔로 운영자 맥락에서 주의

**규모의 함정**: 배민이 3개월 걸린 메뉴뚝딱AI는 20여 명 팀이 만든 것입니다. 솔로로 같은 범위를 추구하다 번아웃이 옵니다. "32% 고민러 잡기"라는 문제 정의만 빌려오고, 구현은 Claude API 5줄로 줄이는 것이 현실적입니다.

**내부 플랫폼의 역설**: 물어보새 같은 내부 에이전트 플랫폼은 임계 사용자 수(critical mass)가 있어야 가치가 납니다. 직원이 1명인 솔로에겐 자기 자신이 유일한 사용자이므로 Obsidian + Claude.ai 조합이 물어보새의 솔로 대응입니다.

## 강의 메모 후보 (Pain/숫자/삽질/훅)

- **Pain**: 배민 앱을 여는 사람의 32%는 "오늘 뭐 먹지?"를 결정 못 한 채로 들어옵니다. 비서앱·ERP를 열 때도 사용자의 32%는 "오늘 뭐 할지" 모릅니다. 이 빈 화면을 AI가 채워야 합니다.
- **숫자**: 물어보새 도입 후 조직 문의 해결 속도 **21% 단축**, 에이전트 긍정 반응률 **95%** — 이 숫자를 만드는 데 대기업 팀이 필요하지 않습니다. 로직 자체는 Claude 한 줄입니다.
- **삽질**: 배민도 3개월 동안 GPT 응답 품질 안정화에 헤맸습니다. 1,000명 튜링 테스트를 돌리고, 리뷰 데이터의 개인정보를 숫자로 치환하는 작업까지 직접 했습니다. '그냥 GPT API 붙이면 되겠지'는 착각입니다.
- **훅**: "한국에서 하루 수백만 건 배달을 처리하는 회사도 AI 에이전트를 도입하는 데 3개월씩 삽질했습니다. 여러분은 그 삽질을 10분 만에 건너뛸 수 있습니다."
