# 조코딩(조동근) — 72만 구독자 신뢰를 기업 AX 전환 플랫폼으로 레버리지하다

## 누구/무엇인가

조코딩은 본명 조동근, 2019년 2월부터 시작한 한국 최대 코딩 교육 유튜브 채널(현 72만+ 구독자)의 운영자입니다. 처음엔 "비전공자도 할 수 있는 코딩"이라는 콘셉트로 파이썬 기초, 웹 개발 등을 다뤘지만, 2025년을 기점으로 AI 에이전트 실전 콘텐츠로 중심축을 완전히 이동했습니다. 2025년 5월 구글 I/O 현장에서 CEO 순다르 피차이와 단독 인터뷰를 진행하며 한국 IT 크리에이터 중 상징적인 지위를 확보했습니다. 2026년 3월에는 유튜브를 넘어 기업 AI 전환(AX: AI Transformation) 컨설팅 전문사 **조코딩AX파트너스**를 설립했습니다. 공동 창업자는 네이버·카카오·당근을 거친 테크리드 출신 문경원으로, 기술 실행력과 채널 신뢰를 결합한 구조입니다. 창업 한 달 만에 대웅제약·호반·슈피겐코리아를 수주하며 빠르게 실적을 냈고, 이제 채용 해커톤·창업 지원·기업 교육까지 AI 생태계를 세로로 통합하고 있습니다.

---

## 무엇이 특별한가

### 1. 콘텐츠가 영업 채널이 된 B2B 전환

72만 구독자 채널은 조코딩의 가장 강력한 영업 자산입니다. 기업이 AX 컨설팅사를 찾을 때 유튜브에서 클로드 코드 영상을 보고 신뢰를 쌓은 조코딩에게 연락하는 구조가 만들어졌습니다. 일반적인 컨설팅사는 영업팀과 제안서에 비용을 쓰지만, 조코딩은 콘텐츠가 그 역할을 무료로 수행합니다. "한 달 만에 대기업 3사 수주"는 이 채널 레버리지의 직접적인 결과입니다. 이는 대표님이 느린호밀 강의로 쌓는 신뢰가 와당탕 비서앱·ERP 도입 문의로 전환될 수 있는 동일한 메커니즘입니다.

### 2. 멀티에이전트 실전 전환 — "한 번 쓰면 못 돌아감"

2026년 5월, 조코딩은 "클로드 코드 + 코덱스 + 제미나이 동시에 굴리는 멀티 에이전트 시스템 (한 번 쓰면 못 돌아감)"이라는 영상을 공개했습니다. 단순 도구 소개가 아니라 Claude Code를 오케스트레이터로, Codex와 Gemini CLI를 워커로 배치해 각 에이전트가 독립적으로 서브태스크를 처리하는 실제 세팅을 그대로 보여줬습니다. 같은 해 4월의 "나의 AI 에이전트 전환기 (w. 클로드 코드, 오픈클로)"에서는 18개의 AI 에이전트를 팀으로 세팅하고 실제 업무를 전환한 사례를 공유했습니다. 이 콘텐츠의 핵심은 *이론이 아닌 삽질 기록*이라는 점입니다. 어떤 상황에서 어떤 에이전트가 무너졌는지, 컨텍스트 한계를 어떻게 우회했는지 날것 그대로 담겼습니다.

### 3. 1인 창업가 프로젝트 — 연봉 4,000만원 + 순이익 10%

조코딩이 설계한 가장 독창적인 구조는 "1인 창업가 지원" 프로젝트입니다. 개발자에게 기본 연봉 4,000만원을 지급하고, 본인이 만든 AI 프로덕트의 월 순이익 10%를 현금 인센티브로 지급합니다. 사무실·맥북·클로드 코드 맥스·커서 구독료·서버비까지 회사가 부담합니다. 이 구조를 공개한 지 2주 만에 200여 명이 지원했습니다. 이는 "솔로 창업의 리스크(수입 불안정, 도구 비용)"를 회사가 흡수하고, "실행의 업사이드"를 개인이 가져가는 하이브리드 모델입니다. 전통적인 직원 고용도, 지분 나누는 공동창업도 아닌 제3의 구조입니다.

### 4. OpenAI × 조코딩 생태계 구축

2026년 3월 조코딩-OpenAI-프라이머 공동 해커톤에서 3,295명이 참가했습니다. 프라이머(초기 투자 전문사)는 우수 팀에게 최대 1억원의 창업 지원금을 별도 제공했습니다. 그리고 7월 18일(이번 주 토요일) **AX 인재전쟁 2026**이 오프라인으로 열립니다. OpenAI × 조코딩AX파트너스 공식 협력으로, 5,000명이 몰렸고 무신사·채널톡·카카오페이증권·마이리얼트립·삼일회계법인 등이 채용 기업으로 참여합니다. 학력·경력·나이·스펙 무관, AI 실력만으로 대기업 채용 라인에 올라가는 구조입니다. 이는 조코딩 채널이 콘텐츠를 넘어 **AI 인재 마켓플레이스**로 진화했음을 보여줍니다.

### 5. AX 아카데미 — 기업 교육의 체계화

조코딩AX파트너스는 AX 아카데미(axacademy.jocodingax.ai)를 별도로 운영합니다. 진단→교육→구축→확산의 4단계 프레임워크로 경영진과 실무자를 대상으로 맞춤형 AI 리터러시 교육을 제공하고, 수료 후 직원들이 직접 자동화 워크플로우와 AI 에이전트를 구축하도록 지원합니다. 콘텐츠(유튜브) → 개인 강의(유데미) → 기업 교육(AX 아카데미) → 기업 컨설팅(AX파트너스) → 채용 이벤트(AX 인재전쟁)로 이어지는 완전한 세로 스택입니다.

## 와당탕/느린호밀 적용 포인트

**오늘 30분 (micro)**:
비서앱에서 "AI 직원 역할카드" 1개를 조코딩식으로 작성해보세요. 형식은 단순합니다 — 이름(예: "리서치 에이전트 우람"), 담당 업무 2줄, 주요 도구 1개. 이 카드를 CLAUDE.md에 추가하거나 비서앱의 에이전트 설정 페이지에 붙이면 됩니다. frontmatter의 `micro_action`과 동일합니다.

**이번 주 1-2시간 (mid)**:
Claude Code + Gemini CLI 또는 opencode(오픈클로) 두 에이전트를 동시에 켜고 동일한 ERP 기능 요청을 각각 보내 결과를 비교해보세요. 코드 스니펫:

```bash
# 터미널 1: Claude Code 오케스트레이터
claude --continue

# 터미널 2: opencode 워커 (오픈클로)
opencode

# Claude Code 프롬프트 예시:
# "아래 기능을 구현해줘. 동시에 opencode에 같은 요청 넣을 거니 두 결과를 비교해서 더 나은 쪽을 채택할 거야."
```

두 결과를 비교하고 "어떤 경우에 어떤 모델이 더 나은가"를 비서앱 메모로 남기세요. 조코딩이 영상에서 말한 대로, 이 비교 데이터가 쌓이면 모델 선택의 직관이 생깁니다.

**이번 달 실험 (macro)**:
느린호밀 수강생 중 관심 있는 분들과 소규모 "AX 실험단"을 운영해보세요. 조코딩의 1인 창업가 모델에서 참고할 구조: 수강생이 비서앱의 특정 기능(예: 레시피 에이전트, 재고 자동화)을 AI로 구현하면 대표님이 실제 사용해보고 피드백을 제공하는 방식. 측정 지표는 ① 완성된 기능 수 ② 비서앱에 실제 통합된 수 ③ 수강생의 구현 시간. 이 과정을 콘텐츠로 만들면 느린호밀의 교육 플라이휠이 됩니다.

## 한국 솔로 운영자 맥락에서 주의

**콘텐츠 채널이 먼저다**: 조코딩AX파트너스가 창업 한 달에 대기업 3사를 수주한 것은 72만 채널이 없었다면 불가능했습니다. 채널 없이 AX 컨설팅 간판만 걸면 평범한 중소 IT 교육업체와 동일합니다. 대표님의 비서앱/ERP 사업도 마찬가지 — 느린호밀의 신뢰 레이어가 기술 제품의 채택을 만들어줍니다.

**AX 인재전쟁 구조 논란**: 이번 해커톤에 대해 일부 커뮤니티에서는 "문제 정의를 참가자에게 떠넘긴다", "취업준비생의 무급 노동으로 기업이 아이디어 풀을 확보한다"는 비판도 있습니다. 해커톤 포맷을 그대로 복사할 경우 참가자-기업-운영자 간 역할 명확화와 지식재산 귀속 문제를 사전에 설계해야 합니다. 이벤트 규모가 클수록 책임 설계가 더 중요합니다.
