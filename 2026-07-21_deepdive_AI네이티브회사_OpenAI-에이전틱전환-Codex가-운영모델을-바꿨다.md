# OpenAI 에이전틱 전환 — Codex가 운영 모델을 바꿨다

## 누구/무엇인가

2026년 6월 말, OpenAI가 자사 직원 전체를 대상으로 한 실증 연구 보고서를 공개했습니다. 제목은 **"The Shift to Agentic AI: Evidence from Codex"** (arXiv:2606.26959). 저자는 Drew Johnston과 David Holtz로, OpenAI 리서치팀이 직접 자기 조직을 해부한 논문입니다.

이 보고서가 특별한 이유는 단순한 기술 발표가 아니라는 점입니다. 외부 플랫폼 사용자(개인·조직 계정)와 OpenAI 내부 직원의 에이전트 사용 데이터를 프라이버시 보호 파이프라인을 통해 비교 분석한 '실측 데이터'입니다. 챗봇 시대가 끝나고 에이전트 시대가 본격화됐다는 선언을 숫자로 증명한 문서입니다.

OpenAI Codex는 2025년 등장한 코딩 에이전트였지만, 2026년 상반기를 지나며 비개발자 부서까지 침투한 범용 에이전트 인프라가 됐습니다. 보고서 발표 시점(2026년 6월) 기준 OpenAI 직원의 **97.9%**가 에이전트를 사용하고 있습니다.

## 무엇이 특별한가

### 1. 99.8% — 에이전트가 챗봇을 먹어치웠다

2026년 6월 11일 기준, OpenAI 직원들이 Codex와 ChatGPT를 합산해 생성한 주간 출력 토큰 중 **99.8%**가 Codex에서 나왔습니다. 사실상 내부에서 챗봇(ChatGPT)은 보조 도구로 전락한 겁니다. 평균 직원 한 명의 출력 토큰도 **85% 이상**이 Codex 경유입니다.

이것이 의미하는 바는 명확합니다. "내가 쓴다"에서 "에이전트가 나를 대신해 실행한다"로 패러다임이 바뀌었다는 것. 쿼리(질문)의 시대에서 위임(delegation)의 시대로.

### 2. 비개발자가 더 빠르게 채택했다 — 137배, 189배

2025년 8월 대비 2026년 현재, 외부 개인 계정의 비개발자 Codex 사용자는 **137배**, 조직 계정 비개발자는 **189배** 증가했습니다. 개발자의 성장 속도보다 **3배 이상 빠릅니다.**

지식 노동자가 현재 Codex 전체 사용자의 약 1/5을 차지합니다. 이들이 Codex로 하는 일은 코드가 아닙니다 — 문서 작성, 데이터 분석, 커뮤니케이션, 법률 검토. OpenAI 법무팀의 경우 2025년 11월 대비 **월간 출력 토큰이 13배** 증가했고, 연구팀은 **50배 이상** 치솟았습니다. Legal, Finance, Recruiting 세 부서 모두 2026년 4월부터 Codex를 '주 도구'로 공식 채택했습니다.

> "The same agentic primitives — multi-step tool use, self-checking, 1M context — apply directly to document-to-PPT workflows and other long-horizon knowledge-work tasks."
> — OpenAI 보고서

즉 개발자를 위해 만든 도구가 모든 지식 노동자의 기본 운영 인프라가 된 겁니다.

### 3. 작업 단위가 바뀌었다 — '요청' 대신 '위임'

2026년 5월 기준, Codex 사용자의 **80.6%**가 30분 이상 걸리는 작업을 위임한 경험이 있고, **70.2%**가 1시간 이상 작업을 맡겼습니다. 8시간 이상 장기 작업을 위임한 사용자 비율은 연초 대비 **거의 10배** 뛰었습니다.

이것이 패러다임 전환의 핵심입니다. 이전 AI 도구의 단위는 '대화 1회전'이었습니다. 이제는 '프로젝트 하나'입니다. 2026년 6월 기준 Codex 턴의 **60.3%**가 외부 도구를 1개 이상 호출합니다. 에이전트가 스스로 웹을 검색하고, 코드를 실행하고, 파일을 수정하며 일을 완결합니다.

### 4. AgentKit — 다음 인프라 레이어

보고서와 맞물려 OpenAI는 **AgentKit**을 발표했습니다. 드래그앤드롭 방식의 시각적 멀티에이전트 워크플로우 빌더(Agent Builder), 데이터·도구 연결 거버넌스(Connector Registry), 제품 임베드용 채팅 인터페이스(ChatKit), 그리고 샌드박스 실행 지원 Agents SDK 업데이트가 포함됩니다. 코드 없이 멀티에이전트 파이프라인을 조립할 수 있는 노코드 레이어가 생긴 겁니다.

## 와당탕/느린호밀 적용 포인트

**오늘 30분 (micro)**: 비서앱에서 오늘 반복 처리한 업무 1가지를 골라 "이걸 Claude에게 8시간짜리 작업으로 위임한다면 어떻게 프롬프트를 짤까?" 1문장 메모를 Obsidian daily note에 추가합니다. 보고서의 핵심 교훈은 '복잡한 작업을 위임해도 된다는 허가'입니다. 위임 근육을 키우는 첫 걸음입니다.

**이번 주 1-2시간 (mid)**: 와당탕/느린호밀의 주간 반복 업무 목록을 뽑아 '대화 1회전으로 끝나는 것' vs '30분 이상 위임 가능한 것'으로 분류합니다. '30분 이상' 카테고리 중 1개를 Claude Code 에이전트에게 실제 위임해봅니다. 예: 주간 매출 데이터를 받아 → 보고서 초안 작성 → Obsidian에 저장 → 텔레그램 발송까지 하나의 태스크로 위임.

```python
# 예시: 비서앱 내 장기위임 에이전트 태스크 구조
task = {
    "title": "주간 매출 보고 에이전트",
    "horizon": "30min",  # 위임 복잡도 태깅
    "steps": [
        "ERP에서 이번 주 매출 데이터 조회",
        "지난 주 대비 증감 계산",
        "Obsidian 형식으로 마크다운 보고서 작성",
        "텔레그램 채널에 요약 발송"
    ],
    "trigger": "매주 월요일 08:00 KST"
}
```

**이번 달 실험 (macro)**: 보고서의 '부서별 채택 모델'을 와당탕 운영에 이식합니다. 비서앱의 업무를 '운영(영업·CS·발주)', '콘텐츠', '재무' 3개 버킷으로 나누고, 버킷별로 에이전트 도입 2주 실험 → 토큰 사용량과 처리 시간을 측정해 OpenAI 법무팀의 '13배 효율화'가 재현되는지 확인합니다.

## 한국 솔로 운영자 맥락에서 주의

**"OpenAI니까 가능하다"는 함정**: OpenAI는 이미 GPT-5급 에이전트를 내부 도구로 쓰고, 에러가 나도 자체 연구팀이 즉시 수정합니다. 대표님이 Claude Code로 비서앱을 만드는 상황은 도구 품질도 다르고, 에러 대응 여력도 다릅니다. 97.9% 채택률을 목표로 삼지 말고, 가장 반복적인 업무 1개부터 위임하는 '작은 성공'을 쌓아야 합니다.

**장기 작업 위임의 감시 비용**: 보고서는 8시간짜리 작업을 에이전트에게 맡기는 것이 가능해졌다고 말하지만, 그 결과를 검토하는 시간은 여전히 인간 몫입니다. 솔로 운영자에게 '에이전트 감시 오버헤드'가 새 병목이 될 수 있습니다. 위임한 작업에는 반드시 '결과 검수 30분'을 일정에 함께 블록하세요.

## 강의 메모 후보 (Pain/숫자/삽질/훅)

- **Pain**: "나는 AI 도구를 매일 쓰는데 왜 업무가 줄지 않을까?" — 이유는 쓰는 게 아니라 '시키는' 게 없기 때문입니다. 대화 1회전으로만 쓰면 챗봇이지, 에이전트가 아닙니다.
- **숫자**: OpenAI 직원의 주간 AI 토큰 99.8%가 Codex(에이전트) 경유. 0.2%만 ChatGPT(챗봇). 비개발자 외부 사용자 137배 성장. 법무팀 월간 토큰 13배, 연구팀 50배.
- **삽질**: OpenAI 보고서가 흥미로운 역설을 지적합니다. 에이전트 사용이 폭발적으로 늘었지만, 가장 어려운 점은 여전히 "무엇을 위임할지 결정하는 것"이었습니다. 도구 능력이 판단 능력보다 먼저 앞서 간 겁니다.
- **훅**: "OpenAI 직원 100명 중 98명이 에이전트를 씁니다. 그런데 그 에이전트한테 8시간짜리 일을 맡깁니다. 대표님은 오늘 AI한테 몇 시간짜리 일을 시키셨나요?"
