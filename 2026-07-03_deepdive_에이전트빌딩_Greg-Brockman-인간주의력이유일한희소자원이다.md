# Greg Brockman — 인간의 주의력이 유일한 희소 자원이다

## 누구/무엇인가

Greg Brockman은 OpenAI 공동창업자이자 현재 사장(President)입니다. 2015년 OpenAI 창업 멤버 중 한 명으로, 초기에는 CTO를 맡았고 Stripe 초기 CTO 경험을 바탕으로 기술 인프라 설계 감각을 OpenAI에 이식했습니다. 2024년 8월 약 3개월의 사바티컬을 떠났다가 11월에 복귀했고, 복귀 직후 인프라 총괄 역할인 'Builder-in-Chief'로 Stargate 프로젝트를 이끌었습니다. 2026년 5월에는 ChatGPT와 Codex 팀을 통합해 OpenAI의 제품 전략 전체를 단독 총괄하게 됐습니다. ChatGPT와 Codex의 주간 이용자 합산이 약 10억 명에 달하는 지금, OpenAI에서 Sam Altman이 외교와 자금 조달을 담당한다면 Brockman은 실제 빌딩을 담당하는 인물입니다.

## 무엇이 특별한가

### 1. "실행은 쉬워졌다 — 이제 부족한 건 판단이다"

2026년 4월 Sequoia AI Ascent에서 Brockman은 이렇게 말했습니다.

> **"Human attention is going to be this incredibly scarce resource. The doing of things now is easy. The 'is this a good thing? Is this what I wanted?' — that is going to become the single most important bottleneck."**
>
> (인간의 주의력은 앞으로 극도로 희귀한 자원이 될 것입니다. 일을 실행하는 것은 이제 쉽습니다. '이게 좋은 것인가? 내가 원한 것인가?'라는 질문이 가장 중요한 병목이 될 겁니다.)

이 발언에서 주목할 지점은 "실행"이 아닌 "판단"을 희소 자원으로 재정의했다는 점입니다. 소프트웨어 개발이든 콘텐츠 제작이든 자동화 흐름이든, AI가 점점 더 많은 실행을 대체할수록 인간에게 남는 경쟁 우위는 "방향성 설정, 의도 명확화, 결과물 검증"입니다.

### 2. AI가 코드 작성 비율을 한 달 만에 20% → 80%로 올렸다

같은 행사에서 Brockman은 OpenAI 내부 수치를 공개했습니다.

> **"We went from these agentic coding tools writing 20% of your code to writing 80% of your code in December alone. They go from being kind of a sideshow to being the main thing that you're doing."**
>
> (에이전트형 코딩 도구가 코드의 20%를 작성하다가 단 12월 한 달 만에 80%까지 올라갔습니다. 부가 기능에서 핵심 작업 방식으로 바뀐 겁니다.)

12월 한 달 동안의 변화입니다. 이 속도는 기존의 기술 도입 곡선과 전혀 다릅니다. 2026년 5월 No Priors 팟캐스트에서는 "OpenAI 코드 중 AI가 쓰지 않은 비율을 추적하기가 어려울 정도"라고 했습니다. Benchmark도, 기준도 역전됐습니다.

### 3. "에이전트 함대의 CEO"가 새로운 사용자 패러다임이다

Brockman은 AI 슈퍼앱(ChatGPT + Codex + Atlas 브라우저를 단일 데스크톱 앱으로 통합)의 비전을 이렇게 설명합니다.

> **"You become the CEO of a fleet of hundreds of thousands of agents that are completing your objectives, your goals, and your vision."**
>
> (당신은 수십만 개의 에이전트 함대를 이끄는 CEO가 됩니다. 에이전트들은 당신의 목표와 비전을 실행합니다.)

이 문장을 그냥 지나치면 안 됩니다. 솔로 운영자에게 이것이 의미하는 바는 매우 구체적입니다. 비서앱·ERP를 Claude Code로 혼자 풀스택 개발한다는 것은 정확히 이 패러다임입니다. 실행 병렬화는 AI가 맡고, 의도와 방향은 본인이 설정하는 구조입니다.

### 4. 컴퓨팅 파워가 경제의 화폐가 된다

2026년 4월 X 포스트에서 Brockman은 이렇게 썼습니다.

> **"The rate and sophistication of AI-driven problem solving will be bound by the amount of compute you have access to."**
>
> (AI 기반 문제 해결의 속도와 정교함은 접근 가능한 컴퓨팅 양에 의해 결정될 것입니다.)

OpenAI의 2026년 컴퓨팅 지출 계획은 $500억(약 68조 원)입니다. 개인 수준에서 무제한 컴퓨팅을 가질 수는 없지만, Claude Code + API + 에이전트 파이프라인으로 접근 가능한 컴퓨팅을 최대한 활용하는 것이 솔로 운영자의 경쟁력 공식으로 이어집니다.

### 5. "우리는 AGI에 70~80% 와 있다, 돌파구 2개 남았다"

No Priors와 MLST 팟캐스트(2026년 5월)에서 Brockman은 이렇게 말했습니다.

> **"OpenAI is 70 to 80% of the way to AGI… we're maybe two breakthroughs away."**
>
> (우리는 AGI의 70~80%에 와 있습니다. 아마 두 번의 돌파구가 남았을 겁니다.)

Big Technology 팟캐스트(2026년 4월)에서는 더 명확히 말했습니다. "AGI가 올 것이냐"는 논쟁은 종료됐으며, 이제는 "언제, 어떻게 활용하느냐"만 남아 있다고.

## 와당탕/느린호밀 적용 포인트

**오늘 30분 (micro)**: 비서앱 또는 옵시디언에서 오늘 처리할 업무 목록을 작성하고 "AI 위임 (실행)" 항목과 "내 집중 (판단)" 항목으로 나눈다. Brockman의 프레임 그대로: 실행은 AI, 판단은 나. `micro_action`과 동일.

**이번 주 1-2시간 (mid)**: 비서앱 ERP에서 반복 루틴 하나(예: 주문 정리, 재고 알림, 고객 응답 초안)를 에이전트 태스크로 변환하는 프로토타입 작성. Brockman의 "에이전트 함대 CEO" 개념을 비서앱 내 실제 버튼 1개로 구현.

```python
# 예시: 비서앱 ERP에서 일일 판단 대기 큐 생성 스니펫
# 에이전트가 실행 완료한 항목을 "판단 필요" 큐에 올리는 패턴
def build_judgment_queue(completed_tasks: list[dict]) -> list[dict]:
    """AI가 실행한 결과 중 인간 승인이 필요한 항목만 필터링"""
    return [
        task for task in completed_tasks
        if task.get("requires_human_review", False)
        or task.get("confidence_score", 1.0) < 0.85
    ]
```

**이번 달 실험 (macro)**: "에이전트가 실행하고 대표님이 판단한다"는 워크플로우를 느린호밀 운영에서 1개 파일럿. 예: AI가 SNS 초안 3개 작성 → 대표님이 1개 선택 → 예약 발행 자동화. 측정 지표: 주간 판단 횟수 대비 실행 건수 비율(목표: 1:10 이상).

## 한국 솔로 운영자 맥락에서 주의

**"에이전트 함대 CEO" 비유를 문자 그대로 받아들이면 위험합니다.** Brockman이 말하는 수십만 개 에이전트는 $500억 인프라 위에서 돌아갑니다. 솔로 운영자가 너무 복잡한 에이전트 파이프라인을 구축하면 관리 비용이 수익을 초과합니다. 핵심은 "함대의 숫자"가 아니라 "판단만 남기고 나머지는 다 위임한다"는 사고방식입니다. 비서앱·ERP 안에서 버튼 1개가 에이전트 1개를 대체하는 구조부터 시작하는 것이 현실적입니다.

**AGI 타임라인 발언은 영업 발언과 믿음 사이 어딘가에 있습니다.** "2개 돌파구 남았다"는 표현은 OpenAI의 기업가치($8,520억 목표)와 IPO 일정과 맞물린 컨텍스트에서 나왔습니다. 솔로 운영자 입장에서 AGI 타임라인보다 중요한 건 "지금 내 업무에서 AI가 80%를 실행한다면 나는 무엇을 할 것인가"라는 자기 질문입니다.

## 강의 메모 후보 (Pain/숫자/삽질/훅)

- **Pain**: 혼자 다 해야 한다는 압박이 솔로 운영자를 짓누릅니다. 실행 하나하나에 집중력을 쏟다 보면 방향을 잃습니다.
- **숫자**: OpenAI 내부에서 AI 코드 작성 비율이 한 달 만에 20% → 80%로 뛰었습니다. 이 속도가 이미 당신 업무에도 적용되고 있습니다.
- **삽질**: Brockman 본인도 2024년 번아웃으로 3개월 사바티컬을 떠났습니다. AGI를 만드는 사람도 지칩니다. 판단 에너지 보존이 전략입니다.
- **훅**: "에이전트가 밤새 코드를 다 짜놨다 — 아침에 일어났더니 프로젝트가 완성됐다." Brockman이 Sequoia에서 직접 한 이야기입니다. 이것이 지금 솔로 운영자가 설계해야 할 업무 구조입니다.
