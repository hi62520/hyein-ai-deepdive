# Lenny Rachitsky 뉴스레터OS — 솔로 크리에이터가 100만 독자를 시스템화한 방법

## 누구/무엇인가

Lenny Rachitsky는 Airbnb에서 호스트 성장을 $100M → $1B ARR 구간에서 이끈 PM 출신이다. 2019년 Substack에서 뉴스레터를 시작해 2025년 3월 100만 구독자를 돌파했으며 현재 세계 최대 프로덕트 뉴스레터·팟캐스트·커뮤니티를 혼자서 운영한다. 구독 수익만 연간 $2M+, 팟캐스트 스폰서십 $500K+ 등 연간 $3M 이상을 솔로로 창출한다. 2025년 이후에는 AI 실용 활용을 집중적으로 다루는 "How I AI" 팟캐스트를 별도로 론칭했고, Claude Code를 비기술자가 쓰는 법부터 Anthropic CPO 인터뷰까지 AI 네이티브 운영의 실험장이 됐다.

핵심은 단 하나의 니치(프로덕트·성장·커리어)에서 6년간 복리로 신뢰를 쌓은 것이다. 모든 글은 "내가 직접 조사했거나 경험한 것만 쓴다"는 원칙 아래 작성되고, 1,750명 이상이 응답한 자체 서베이부터 Anthropic·Shopify·Stripe 내부자 인터뷰까지 직접 데이터를 생산한다.

## 무엇이 특별한가

**1. 100만 독자 = 솔로 리서치 인프라**

2025년 3월 100만 구독자 달성 직후 Lenny는 "이 커뮤니티가 나의 가장 큰 자산"이라 밝혔다. 예를 들어 "AI 도구로 vibe coding한 것 중 실제 쓰고 있는 게 뭐야?"라는 단순한 질문 하나가 X·LinkedIn·뉴스레터 슬랙에서 **24시간 안에 1,000개 이상의 답변**을 끌어냈다. 아파트 배달 자동 응답기, 혈당 관리 앱(Replit으로 제작, 9개월 만에 85,000 유저), 개인 맞춤 언어 학습 뉴스레터 등 다양한 예시가 등장했다. 이 데이터를 편집해 하나의 아티클로 만드는 것이 전형적인 Lenny OS다. 구독자가 곧 조사원이자 편집 기여자가 된다.

**2. AI 생산성 서베이 — 창업자가 가장 큰 수혜자**

2025년 진행된 대규모 AI 생산성 서베이(1,750명 응답)에서 나온 수치는 인상적이다. "AI가 기대를 초과했다" 55%, "업무 품질이 개선됐다" ~70%, "AI로 주당 최소 반나절 이상 절약" 절반 이상. 그리고 **창업자/솔로프레너의 49%가 주당 6시간 이상 절약한다고 응답했으며 이는 어떤 직군보다 높은 수치다.** ("Founders are getting the most out of AI. Half (49%) report that AI saves them over 6 hours per week, dramatically higher than for any other role.") 솔로 사업가가 AI 활용 ROI에서 가장 유리한 위치에 있다는 실증 데이터다. 단, 응답자의 92.4%가 적어도 한 가지 단점도 경험한다고 답해, 장밋빛 과장은 경계한다.

**3. "Claude Code를 Claude Local로 생각하라" — 비기술자 재프레이밍**

Lenny의 가장 화제가 된 2026년 아티클 제목은 **"Everyone should be using Claude Code more"**다. 핵심 주장: "The trick is to forget that it's called Claude Code and instead think of it as Claude Local or Claude Agent. It's essentially a super-intelligent AI running locally, able to do stuff directly on your computer." (번역: 핵심 요령은 이름에서 'Code'를 지우고 'Claude Local' 또는 'Claude Agent'로 재인식하는 것이다. 로컬에서 직접 작동하는 초지능 AI라는 관점.) PM·마케터·디자이너·창업자·부모까지 모두를 대상으로 포지셔닝하면서, 파일 정리부터 고객 통화 요약, Linear 티켓 생성, 이미지 품질 개선까지 50가지 비기술자 활용 사례를 실제 예시와 함께 정리했다.

**4. "How I AI" — AI 워크플로우의 정기 미디어화**

2025년 론칭한 팟캐스트 "How I AI"(진행: Claire Vo)는 30분짜리 에피소드에서 게스트가 **화면 공유를 통해 실제 AI 워크플로우를 라이브 시연**한다. Stripe의 프로토타이핑 내부 툴, Notion 디자인팀의 Claude Code 프로토타이핑, Anthropic 디자인 툴 실시간 비교 등이 다뤄졌다. 강의나 이론이 아닌 "바로 복사해서 쓸 수 있는 구체적 팁"이 포맷의 핵심이다. 콘텐츠-커뮤니티-자동화 파이프라인을 미디어 브랜드 안에 구현한 구조다.

**5. Anthropic 내부자 인터뷰 — 코드의 90%가 AI가 씀**

2026년 Lenny가 Anthropic CPO Mike Krieger와 나눈 인터뷰에서 나온 수치: **"Claude Code now writes 90-95% of Claude Code"** — Claude Code 자체 코드의 90~95%를 Claude Code가 작성한다. Lenny의 코멘트: "코드 작성 병목은 사라지고, 병목이 의사결정(무엇을 만들지)과 머지 큐(코드를 어떻게 통합할지)로 이동했다." 솔로 개발자에게도 동일한 구조 변화가 일어나고 있다는 증거다.

## 와당탕/느린호밀 적용 포인트

**오늘 30분 (micro)**: 옵시디언에 "What I AI" 노트를 만들고 Lenny 서베이 문항 그대로 적용한다 — "AI로 지난주 가장 시간을 절약한 작업 Top 3는?" 적어보고, 각 항목 옆에 '비서앱 자동화 가능 여부'를 O/X로 표시. 이것이 다음 Claude Code 스프린트의 우선순위 입력값이 된다. frontmatter `micro_action`과 동일.

**이번 주 1-2시간 (mid)**: "How I AI" 포맷을 와당탕연구소 콘텐츠에 접목한다. 본인이 Claude Code로 비서앱/ERP를 실제 사용하는 화면을 5분짜리 영상으로 녹화하고, 해당 프롬프트를 공개한다. 이것이 가장 진입 장벽 낮은 AI 실용 콘텐츠다. 기술: OBS 로컬 녹화 + 옵시디언에 프롬프트 템플릿 저장.

```markdown
## 오늘 내가 AI로 한 일 (How I AI 템플릿)
- 작업: [내가 한 것]
- 도구: Claude Code / 비서앱
- 소요 시간: [실제] → [AI 이전 예상]
- 재현 가능한 프롬프트: [붙여넣기]
- 다음에 개선할 점: [한 줄]
```

**이번 달 실험 (macro)**: Lenny 뉴스레터의 "자체 서베이로 콘텐츠 생성" 모델을 작게 복제한다. 와당탕연구소 뉴스레터/채널에서 독자에게 질문 하나를 던지고("AI로 가장 놀란 경험?"), 30~50개 답변을 모아 편집한다. 측정 지표: 응답 수, 해당 아티클 오픈율·공유수. 성공 기준: 응답 30개 이상 + 오픈율 기존 평균 +5%p.

## 한국 솔로 운영자 맥락에서 주의

**규모 차이를 직시할 것.** Lenny의 "커뮤니티가 곧 리서치 인프라" 전략은 100만 독자가 있어서 작동한다. 1만 미만 독자 규모에서 서베이를 돌리면 통계적 신뢰도보다 관계 기반 신뢰가 더 중요하다. 질문 하나를 SNS에 올리는 것보다, 10명에게 직접 DM으로 물어보는 방식이 오히려 더 깊은 통찰을 준다.

**비즈니스 모델 맥락이 다르다.** Lenny의 수익 구조는 영어권 글로벌 구독자 + Substack + 포드캐스트 스폰서십이다. 한국어 시장에서 $15~30/월 유료 뉴스레터는 아직 심리적 장벽이 높다. Lenny 모델의 핵심(특정 니치에 집중·직접 데이터 생산·커뮤니티 참여 구조)만 빌리고, 수익화는 강의·컨설팅·SaaS로 먼저 검증하는 것이 현실적이다.
