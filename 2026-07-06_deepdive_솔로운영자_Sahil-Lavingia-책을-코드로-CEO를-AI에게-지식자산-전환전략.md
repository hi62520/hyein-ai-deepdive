# Sahil Lavingia — 회사 CEO에 AI를 앉히고 IRS로 간 창업자: 2026년 가장 과격한 솔로프리너 실험

## 누구/무엇인가

Sahil Lavingia(@shl)는 Gumroad(현 Anti-Work)의 창업자입니다. 18세에 Pinterest의 두 번째 직원으로 일했고, 2011년 Gumroad를 창업했습니다. 크리에이터가 디지털 제품을 직접 판매할 수 있는 플랫폼으로, 2023년 기준 연매출 $20.7M(약 290억 원), 순이익 $8.9M(약 125억 원)을 기록했습니다 — 상근직원 0명으로. 같은 해 크리에이터들에게 지급한 수익은 $247M에 달합니다.

오늘 기사는 그의 2025-2026년 가장 과격한 실험들에 집중합니다. 이 글을 쓰는 2026년 7월 6일 기준, 그는 이미 이런 일들을 마쳤습니다: 회사 CEO에 AI 에이전트를 임명하고, DOGE에서 55일간 자원봉사 엔지니어로 일하다 해고되고, IRS(미국 국세청)에 취업했다가 7월 1일 퇴직했습니다. 그리고 베스트셀러 책을 Claude Code 슬래시 커맨드 10개로 변환해 오픈소스로 공개했습니다.

## 무엇이 특별한가

### 1. 2026년 4월 1일 — AI 에이전트를 CEO로 임명하다

만우절이었지만, 진짜였습니다.

> "Gumroad has a new CEO! Effective today, I'm appointing a new CEO of Gumroad. The new CEO is @Gumclaw, an AI agent."
> — x.com/shl/status/2039401205311398027

Gumclaw가 첫날 한 일: 고객 지원 티켓 251개 처리(기존 인간 팀 전체보다 많음). 첫 주에 206개 커밋, 13개 불안정 테스트 수정 — 인간이 작성한 테스트 코드 한 줄 없이. Gumclaw는 이제 채용 공고도 직접 올립니다. "디자인 엔지니어 구합니다 — 분기당 2주, 뉴욕 근무, 연봉 상당 $189K."

이것이 그가 수년간 쌓아온 AI-first 아키텍처의 자연스러운 끝점입니다. 2025년 2월에 이미 이렇게 선언했습니다:

> "No longer hiring junior or even mid-level software engineers. Our tokens per codebase: Gumroad: 2M / Flexile: 800K / Helper: 500K / Iffy: 200K / Shortest: 100K."
> — x.com/shl/status/1887484068075274347

코드베이스를 토큰 수로 관리하고, Devin AI 에이전트가 PR의 41%를 담당하며(4개월간 1,583개 PR, 85% 이상 머지율), 코드는 자동으로 배포됩니다. Gumclaw는 그 위에 앉은 경영자 레이어입니다.

### 2. GOD Mode → 연쇄 위임의 실행 사례

2024년 10월, Paul Graham의 '창업자 모드' 에세이가 화제를 모으자 Sahil은 *GOD Mode*로 응수했습니다. 그의 주장: 창업자 모드에서 가장 중요한 것은 'Destruction(파괴)' 단계인데, 이는 전체 시스템을 머릿속에 담고 있는 단 한 사람만이 실행할 수 있다는 것입니다.

그리고 그는 2025-2026년에 이것을 세 겹으로 실행했습니다.

- 2025년 4월: Gumroad와 Flexile 소스코드를 MIT 라이선스로 전면 오픈소스화
- 2025년 11월: 14년 만에 Gumroad CEO 자리를 Ershad Kunnakkadan에게 넘기고 Chairman으로만 남음
- 2026년 4월: 새 인간 CEO 위에 AI 에이전트 Gumclaw를 "CEO"로 임명

오픈소스화는 코드베이스를 모두에게 위임한 것이고, 인간 CEO 선임은 운영을 위임한 것이고, AI CEO는 의사결정 레이어 자체를 기계에 맡긴 것입니다. 창업자는 이제 비전과 IP 설계만 남습니다.

> "If you're a software engineer, pivot to being CEO."
> — x.com/shl/status/1991139820190535855 (2025년 11월)

### 3. 책이 실행 코드가 됐다 — 지식 자산의 새로운 형태

2026년 3월 23일, Sahil은 `github.com/slavingia/skills`를 공개했습니다. *The Minimalist Entrepreneur*의 전체 방법론을 10개 Claude Code 스킬로 변환한 것입니다.

스킬 목록:
- `/find-community` — 타겟 커뮤니티 발굴
- `/validate-idea` — 시장성 검증
- `/mvp` — MVP 범위 정의
- `/processize` — 코드 전에 수동 프로세스 먼저
- `/first-customers` — 첫 100명 확보
- `/pricing` — 가격 설계
- `/marketing-plan` — 콘텐츠 마케팅으로 오디언스 구축
- `/grow-sustainably` — 수익성 유지하며 성장
- `/company-values` — 채용 전 문화 정립
- `/minimalist-review` — 모든 사업 결정의 체크리스트

공개 첫 주: 459K 뷰, 5.8K 북마크, 4,800+ GitHub 스타. 책을 읽고 독자가 알아서 적용하던 시대에서, AI가 책의 논리를 내 상황에 맞게 직접 실행하는 시대로의 전환입니다.

### 4. DOGE 55일 + IRS 8개월 — 솔로프리너가 정부에 들어간 이유

2025년 3월 17일, DOGE(정부 효율화 부서)에 무급 자원봉사 엔지니어로 합류했습니다. VA(재향군인부) 계약서를 LLM으로 분석해 낭비를 찾아내는 도구를 만들고 $1.6B 규모의 삭감 후보를 식별했습니다. 하지만 코드를 프로덕션에 배포하는 승인을 끝내 받지 못했습니다. 55일 후 Fast Company 인터뷰가 언론에 공개된 바로 다음 날, 접근 권한이 박탈됐습니다.

NPR 인터뷰에서 그는 "정부가 생각보다 비효율적이지 않았다"고 했습니다 — 효율을 자르러 간 사람치고 이례적으로 솔직한 발언입니다.

이후 같은 해 11월, DOGE 에세이를 읽은 Sam Corcos(재무부 CIO)의 연락으로 IRS에 경력직 직원으로 합류했습니다. 온라인 계좌 팀에서 종이 서류로만 접수되던 사기 신고를 디지털 제출로 전환하는 기능을 실제로 배포했습니다. 그리고 2026년 7월 1일, 8개월 만에 퇴직을 선언했습니다.

> "Excited to retire from the IRS!"
> — x.com/shl/status/2072431305287942482 (2026년 7월 1일)

그가 이 모든 것을 한 이유는 명확합니다: "AI가 일을 가져가는 세상에서 정부 시스템이 어떻게 작동하는지 직접 보고 싶었다."

## 와당탕/느린호밀 적용 포인트

**오늘 30분 (micro)**: x.com/shl 에서 Gumclaw 관련 최근 트윗 3개 읽고, 비서앱에서 내 반복 업무 중 'AI에게 위임할 수 있는 것' 하나를 메모합니다. "고객 문의 1차 응답", "주문 확인 알림", "재고 체크" 중 하나를 고르면 됩니다. `micro_action`과 동일합니다.

**이번 주 1-2시간 (mid)**: `github.com/slavingia/skills`를 클론해서 `/processize` 스킬을 느린호밀 주문 처리 플로우에 적용해봅니다. "코드 전에 수동 프로세스를 3번 해본다" 원칙은 ERP 기능 추가 시 특히 유용합니다.

```bash
# Claude Code에 스킬 설치 예시
git clone https://github.com/slavingia/skills ~/.claude/skills/minimalist-entrepreneur
# ~/.claude/settings.json 에 skills 경로 추가 후 /processize 실행
```

**이번 달 실험 (macro)**: 와당탕 운영 매뉴얼의 핵심 프로세스 하나를 Claude Code 스킬로 변환합니다. '고객 상담 원칙'이나 '발주 판단 기준'을 `/wadangtang-consult` 형태로 만들면, AI가 내 방식대로 판단하는 실행 가능한 지식이 됩니다. 측정 지표: 스킬 호출 횟수, 해당 업무 처리 시간 단축율.

## 한국 솔로 운영자 맥락에서 주의

**"AI CEO"는 Gumclaw가 수년간 쌓인 코드베이스·데이터·프로세스 위에 올라간 것**입니다. 아직 운영 시작 단계의 비서앱·ERP라면 AI에게 의사결정 레이어를 넘기기 전에 인간이 만든 명확한 규칙과 데이터가 먼저 쌓여야 합니다. Gumclaw가 첫날 251개 티켓을 처리할 수 있었던 것은 Gumroad의 14년 지식이 코드와 데이터에 이미 녹아 있었기 때문입니다.

**오픈소스화는 해자의 포기가 아니라 재정의**입니다. Sahil이 Gumroad 소스코드를 공개한 것은 코드베이스가 더 이상 경쟁 우위가 아니기 때문입니다. 오히려 오픈소스로 커뮤니티를 끌어들이고 바운티($119,400)로 외부 기여를 유발합니다. 한국 솔로 운영자는 '내 코드가 핵심 비밀'이라는 생각에서 벗어나 '내 운영 철학과 고객 관계가 진짜 해자'임을 인식해야 합니다.
