# 지피터스 뽀짝이 사례 → 와당탕/느린호밀 적용 인사이트

원본 영상 노트: [[2026-04-19_지피터스_뽀짝이_오픈클로_AI네이티브]]

## 1. 사수-부사수 멀티 에이전트 구조
지금은 Claude Code 한 세션과 1:1로 일하는 구조 → 뽀야(사수)·뽀짝이(부사수)처럼 분리하면 병목 해소.

- **운영 부사수**: ERP/주문/CS 담당
- **비서·콘텐츠 부사수**: 글쓰기·일정·강의 메모 담당
- **사수 에이전트**: 검수·인수인계 담당
- 부사수가 매일 업무일지 → 사수 리뷰 → 대표는 결재만

## 2. CS 격리 에이전트 (보안 분리)
뽀짝이-익스터널 사례가 즉시 유용.
- 카드/결제/세무 데이터는 메인 에이전트
- 고객 응대용은 별도 워크스페이스에 격리
- **영수증/카드엑셀 매칭** 작업도 "세무사 전용" 격리 에이전트로 → 인젝션 리스크 차단

## 3. 비서앱 = 슬랙 역할로 격상
지피터스가 슬랙을 에이전트 호출 인터페이스로 쓰듯, **hyein-daily 비서앱을 "에이전트 호출창"**으로 진화.
- 메모/포스트잇(다음 TODO)에 `@보자기 OO 보내줘` 같은 멘션 트리거
- 비서앱이 단순 일정관리에서 → 운영 명령 인터페이스로

## 4. 매일 자동 업무일지 + 강의용 메모 4항목
이미 devlog 포맷(Pain/숫자/삽질/훅) 규칙 있음.
- cron으로 매일밤 에이전트가 초안 작성
- 사수 에이전트 검수
- 옵시디언/노션/깃 3중 저장
- 강의 콘텐츠 자동 누적 → 강의 사업 자산화

## 5. 워크스페이스 git 레포 백업
뽀짝이가 메모리 파일을 자기가 줄여버린 사고 사례.
- Claude 메모리 폴더(`/Users/hyein/.claude/projects/...`)를 별도 사설 깃 레포로 백업
- DB 3중 백업과 같은 원칙: 에이전트 메모리도 백업 대상

## 6. 스킬 라이브러리화
반복 작업을 스킬로 구워두면 비서앱·ERP·세무용에서 공통 호출.
- 알림톡 발송
- 주문 동기화
- 정산
- 카드매칭
- 한 번 잘 정리한 스킬이 복리로 시간 절약

## 7. 줌/회의 VTT 교차분석
강의·미팅 녹화에서 텍스트 + 참석률 교차 분석 → 다음 강의 주제 추천.
- 대표 강의 사업 확장 시 직접 적용 가능
- 인기 주제 자동 추출
